# アニメ制作ワークフローとこのツールの位置づけ

## 前提：なぜ静止画が必要か

クリスタはアニメーション制作ツールだが、工程の多くは静止画にする必要がある。

- 仕上げ工程はクリスタを使わないことが多い
- 撮影工程はそもそも静止画の合成
- 次工程が紙（印刷）の場合もある
- 外部スタッフがクリスタを持っていない場合もある

つまり **「クリスタで作ったアニメーション素材を、静止画として正しく取り出す」** というニーズは制作フローの構造的な問題として存在する。

---

## 現場の現実：別トラック方式

クリスタのアニメーション書き出しはタイムシートに打たれた「セル単体」を出力する設計になっている。
そのため、原画・演出修正・作監修正など複数の工程が絡む場合、現場では以下のような構造をとる。

```
レイアウト/
  LO/
    A/
      1番  2番  3番  ...

  演出修正/
    A/
      1番（修正あり）  3番（修正あり）  ...

  作監修正/
    A/
      2番（修正あり）  ...
```

修正が発生したセルに、修正工程のタイムライン上の同じコマに別トラックでセルを置く。
クリスタの書き出しで自動化でき、静止画として次工程に渡せる。

これは、出力の自動化を見据えると効率的だが、紙の作業工程の再現という観点でみると、直観的ではない。タイムシートの複製が煩雑であり、修正指示が同一のレイヤーフォルダで管理できなくなってしまう。

---

## 理論上の理想：セル内包型

各工程をセルの中に従属させる構造。

```
レイアウト/
  A/
    1/
    _作画素材(線画レイヤー等)
    _作画素材(影付レイヤー等)
    _演出修正/
      ...
    _作監修正/
      ...
```

構造として直感的だが、クリスタのアニメーションセル書き出しでは、工程ごとの独立した出力が自動化できない。全工程を合成した画像が1枚出るだけで、修正指示を単独で確認することができない。これでは紙の時の修正用紙が重なるという工程の再現にならない。また、作業途中に視認性のため、別工程の素材の不透明度を下げた状態で保存していると、その影響が出力に混入する。

---

## このツールが解決すること

両方のワークフローに対応し、どちらで作られたデータからも工程ごとの完成画像を正しく取り出せる。

**別トラック方式への対応**
- XDTSまたはCSPBからタイムライン情報を読み込み、修正工程のフォルダ名を登録し、出力時に_en等でファイル名に自動サフィックス付加

**セル内包型への対応**
- セル内の工程フォルダ名を登録することで、それを検出して独立出力
- 工程フォルダ本体は不透明度を自動的に100%として出力、内部の作業レイヤーの不透明度は尊重
- 各工程を単独で確認できる完成画像として出力（紙工程の修正用紙一枚を再現）

**共通の前提**
- アニメーションフォルダ・セルフォルダは素材コンテナの「構造」として、出力時に不透明度を100%として扱い出力する。そのフォルダの中身の不透明度は尊重する。（うっかり作業中に薄く表示するためにレイヤーフォルダの不透明度さげてても大丈夫、でも影付レイヤーの不透明度は勝手にかわらない）
- 作業者が参考素材を半透明にして作業した状態でも、出力は「1枚の紙に描いた」状態になる
