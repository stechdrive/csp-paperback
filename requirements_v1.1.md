# 要件定義書 v1.1

## 概要
CSPのアニメーションセル出力の不備を補う、PSDベースのWebアプリ。gh-pagesで静的ホスティング。ブラウザ完結でサーバー不要。

---

## 入力
- PSDファイル（必須、1カット1ファイル）
- xdtsファイル（オプション）
  - あればアニメーションフォルダの自動特定に使用
  - なければユーザーがUIでアニメーションフォルダを手動指定
- CLIPファイル（開発目標、オプション）
  - SQLiteからタイムシートを解析してxdtsを内部生成
  - 生成したxdtsをPSDのXMPメタデータに埋め込み
- ファイル読み込み方法：ドラッグ＆ドロップ／ファイル開くボタン両対応

---

## アニメーションフォルダ特定

### xdtsあり（または内部生成）
- xdtsのtrackリストからCAMフィールドを除外
- 残ったtrack名でPSDレイヤー名を照合
- ヒット = アニメーションセルフォルダ確定
- PSD書き出し時にレイヤー名が変わっていないことはユーザー責任

### xdtsなし
- ユーザーがレイヤーツリー上でアニメーションフォルダをクリックして手動指定

---

## プロジェクト状態の保存

### XMPメタデータへの埋め込み
- PSDのXMPメタデータ（カスタムネームスペース）にプロジェクト状態を埋め込む
- PhotoshopもCSPも未知のメタデータは無視するためPSD互換性を維持
- PSD1ファイルで完全完結、サイドカーファイル不要

### 保存対象
- xdtsデータ（CLIPから生成した場合）
- シングルマーク情報
- 仮想セット定義
- プロジェクト設定テーブル

### 一時的なステート（保存不要）
- 表示非表示の切り替え状態
- プレビュー状態

### シングルマーク・仮想セットのPSD書き戻し
- シングルマーク・仮想セットはPSD上にレイヤーを生成して永続化
- 特定のネーミングルール（例：`SMARK_レイヤー名`、`VSET_セット名`）で識別
- xdtsにもtrackとして登録
- 次回読み込み時に自動認識

## 合成エンジンの基本思想

### 「一枚の紙に戻す」
静止画出力はデジタル作業中の透かし表示を取り除き、一枚の紙に印刷された状態を再現する。

- **合成モードはキープ**：どう重ねるかの情報なので尊重する
- **不透明度は無視（100%として扱う）**：作業用の透かし設定は出力に関係ない
- 紙スキャン素材が白地込みで入ってきても、乗算や合成モードで処理するCSP上の見た目がそのまま出力に反映される

### フラット化ロジック

全要素をルートまで持ち上げてから合成する。

```
1. ルート直下の全要素を再帰的にフラット化
   - 通常レイヤー → そのまま（合成モードキープ、不透明度100%）
   - レイヤーフォルダ → 中身を合成して単独レイヤーに（合成モードキープ、不透明度100%）
   - アニメーションフォルダを含む親フォルダ → フォルダのままルートまで進出（例外）

2. アニメーションフォルダを含む親フォルダは内部でセルを先に合成
   （合成モードキープ、不透明度100%）

3. ルート直下に単独レイヤーとして並べた状態で合成モードで合成
```

### ルート直下の最終合成イメージ

```
単独レイヤー（レイアウト用紙・Pass Through）
単独レイヤー（撮影指示）
単独レイヤー（セルフォルダ・フラット化済み・合成モードキープ）
単独レイヤー（背景原図）
↓
通常のPSD合成処理をそのまま適用
```

### 合成モードの派閥対応

```
ベタ塗り乗算派
    セルフォルダ → 通過
    中のベタ塗り → 乗算
    → フラット化後、通過で貫通してベタ塗り乗算が下と合成

工程フォルダ合成モード派
    ENフォルダ → 乗算
    中身 → 通常
    → フラット化後、乗算の単独レイヤーとして合成
```

どちらもCSP上での見た目がそのまま出力に反映される。

### 唯一の特別ルール
**アニメーションフォルダを含む親フォルダはフォルダのままルートまで持ち上げる**

これ以外は全部通常のPSD合成ロジックで解決する。

## 合成ルール

### 大前提
- 非表示レイヤー／フォルダは問答無用で合成不参加・出力対象外
- **レイヤーの不透明度は無視（100%として扱う）**
- クリッピングマスクは有効
- 調整レイヤーはag-psdが正しく書き出していれば正しく作用させる
- **合成モードはすべてキープ**

### レイヤー表示非表示による出力制御（3段階）
- 出力したい・合成にも参加させたい → 通常以外の合成モード
- 出力したい・合成には参加しない → `_`プレフィックスをつける
- 出力したくない（一時的） → 目玉オフ（非表示）

---

## アニメーションフォルダの動作モード

アニメーションフォルダごとにモードを選択。

### 通常モード
- アニメーションフォルダ直下の子が1セル
- 子がフォルダでも中身を全合成して1枚に潰す

### セル内包型モード（推奨）
厳格な運用ルールが必要なモード。選択時にUI上で運用ルールを明示表示。

**前提となる運用ルール**
- アニメーションフォルダ内のセルは必ずフォルダ
- セルフォルダ内のセル本体レイヤーは不透明度100%固定（アプリ側で無視するが運用として統一）
- 修正工程フォルダの命名規則をプロジェクト設定テーブルで定義
- コンポジット確認はCSP上で行う
- 修正工程の組み合わせ出力は行わない

**セルフォルダ内の扱い**
- プロジェクト設定テーブルに一致するフォルダ → 単独で `セル名_サフィックス.jpg/png` として出力
- それ以外のフォルダ・レイヤー → セル本体として全合成して `セル名.jpg/png` に

## プロジェクト設定テーブル

工程フォルダ名の読み替えとモード設定をプロジェクトごとに定義。
ローカルに保存、共有用にJSONエクスポート／インポート対応。

### テーブル構造（逆引き方式）

```
サフィックス  → フォルダ名リスト（複数登録可）             → モード
_en         → [EN, 演出修正, ens, 演修]                  → 工程フォルダ／セル内包型
_ka         → [KA, 作監, 作画監督修正, sakkan]            → 工程フォルダ／セル内包型
_ss         → [SS, 総作監, 総作画監督修正]                 → 工程フォルダ／セル内包型
_do         → [DO, 動検, 動画検査]                        → 工程フォルダ／セル内包型
```

- 同一フォルダ名が複数サフィックスに登録された場合はUI側でエラー表示
- テーブル未登録のフォルダ → デフォルトモードで処理、サフィックスなし
- **デフォルトモード：セル内包型（推奨）**
- 工程テーブルのデフォルト値：空（標準セットは後日定義予定）

### 連番桁数設定
- プロジェクトごとに桁数指定（デフォルト4桁）

## 出力モード選択

### 全出力モード
- アニメーションフォルダ → セル連番出力
- アニメーションフォルダを含まない通常フォルダ → 全部出力
- 非表示は除外
- 初回や構造把握に使う、不要なものは後で捨てる運用

### マーク指定モード
- 明示的にマークしたものだけ出力
- 合成参加ルールは全出力モードと共通

---

## マーク機能

### _プレフィックス自動マーク
- フォルダ名が `_` で始まるフォルダを自動でシングルマーク対象とする
- 任意階層で有効
- アニメーションフォルダ内は例外として無視
- 親フォルダに `_` がついていても子フォルダに `_` をつければ個別にも出力される

### シングルマーク
- 単独レイヤーまたはアニメーションフォルダを含まないフォルダに1クリックで付与
- その階層位置のまま1枚出力
- 内部的には1枚セルの仮想アニメーションフォルダとして処理
- アニメーションフォルダを含むフォルダはUI側でグレーアウト
- ホバー時に「仮想セットを使用してください」と表示
- PSDにレイヤーを生成してXMPに情報を埋め込み永続化

### 仮想セット
- 任意の名前をつけて作成
- 差し込み階層位置をドロップで指定
- メンバー（単独レイヤー、フォルダ、特定セル）をドロップで追加
- 差し込み階層位置にアニメーションフォルダを指定した場合は全セルに自動展開して書き出し
- 差し込み階層位置に基づき合成参加ルールが自動適用
- まとめて1枚出力
- 内部処理はシングルマークと統一（メンバー1つの仮想セット）
- PSDにレイヤーを生成してXMPに情報を埋め込み永続化

---

## ファイル出力

### 出力フォーマット
- JPG（デフォルト）/ PNG を選択
- JPG選択時は品質設定あり

### 背景合成
- **白ベタ合成**（デフォルト、JPG選択時は強制）
- **透明**（アルファチャンネルあり、PNG選択時のみ選択可）
- JPG選択時は透明が選べないよう自動連動

### ファイル命名
- アニメセル通常モード：`フォルダ名_セル番号.jpg/png`（例：`A_0001.jpg`）
- アニメセルセル内包型モード：`セル名.jpg/png` / `セル名_サフィックス.jpg/png`
- シングルマーク・_プレフィックス：`レイヤー名.jpg/png`（先頭の_は除去）
- 仮想セット：`セット名.jpg/png`
- 名前被り時：`_2`, `_3` サフィックス自動付加
- 連番桁数：プロジェクト設定に従う（デフォルト4桁）

### 出力構造モード
- 階層保持モード：フォルダ構造をそのままディレクトリに反映（デフォルト）
- フラット展開モード：ルートに全部並べる
- ZIPでダウンロード（ファイル名：{PSDファイル名}.zip）

```
┌──────────┬──────────────┬──────────────┐
│ 仮想セット│   プレビュー  │ レイヤーツリー│
│  管理(左) │   (中央)     │    (右)      │

```

### モバイル対応
- 目指す方向だがUIレイアウトは要検討

### 言語
- JP / EN 対応（ブラウザ言語に従う）

### プレビュー動作
- レイヤーツリー上のアニメーションセルをクリック → プレビュー自動更新
- マーク済み素材をクリック → プレビュー自動更新
- 仮想セットのメンバーを変更 → プレビュー自動更新
- 合成参加ルールを自動適用した結果を表示

### レイヤーツリー（右ペイン）
- PSDのレイヤー階層構造を表示
- アニメーションフォルダは識別表示
- xdtsなし時はアニメーションフォルダを手動指定可能
- _プレフィックスフォルダは自動マーク済みとして識別表示
- アニメーションフォルダごとに動作モード選択（通常／セル内包型）
- セル内包型モード選択時に運用ルールを明示表示
- シングルマーク付与ボタン
- アニメーションフォルダを含むフォルダはマークボタンをグレーアウト
- 各レイヤー／フォルダに目玉アイコンで表示非表示切り替え
- 切り替えはプレビューおよび出力に反映
- ペイン上部に「初期状態に戻す」ボタン（PSD読み込み時の表示状態を保持・復元）

### 仮想セット管理（左ペイン）
- セット作成ボタン
- セット名入力
- 差し込み階層位置ドロップゾーン
- メンバードロップゾーン
- セット一覧表示

### プロジェクト設定
- テンプレート選択（工程フォルダ型／セル内包型）
- 工程テーブル編集（サフィックス・フォルダ名リスト・モード）
- 連番桁数設定
- デフォルトモード設定（デフォルト：セル内包型）
- JSONエクスポート／インポート（スタジオ内での設定共有用）
- ローカル保存

## テスト方針
- CSPで手動書き出しした結果をゴールデンデータとして画像比較
- ゴールデンデータはCSPの不具合が混入しない構成で作成
- Phase1: コア合成エンジン（フラット化ロジック）→ ゴールデンデータでテスト
- Phase2: マーク機能（シングルマーク・_プレフィックス・仮想セット）
- Phase3: UI・出力（階層保持／フラット展開・ZIP出力）
- Phase4: セル内包型モード・プロジェクト設定テーブル
- Phase5: CLIPファイル解析・XMP埋め込み
